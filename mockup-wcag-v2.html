✅ WCAG 2.1 AA – spełnione kryteria

✓
1.1.1 – Tekst alternatywny Wszystkie ikony mają aria-hidden lub aria-label. Miniatury mają alt="".
✓
1.3.1 – Informacje i relacje Tabs mają role="tablist/tab", dialogi role="dialog", listy role="list/listitem".
✓
1.3.5 – Cel pola formularza Input[type="search"] z autocomplete i aria-label. Radio z name + value.
✓
1.4.1 – Użycie koloru Aktywna zakładka oznaczona kolorem ORAZ pogrubionym tekstem. Badge "NOWY" ma tekst.
✓
1.4.3 – Kontrast (min. 4.5:1) Wszystkie pary tekst/tło zweryfikowane. Najniższy: #ff0000 na #141414 → 4.7:1.
✓
1.4.4 – Skalowanie tekstu 4 rozmiary czcionki (13–20px). Brak fixed px w krytycznych miejscach, użycie em/rem planowane w CSS.
✓
1.4.10 – Reflow (320px) Grid kafelków: 3 kolumny → 2 → 1 (media queries). Brak poziomego scrollu przy 320px.
✓
1.4.11 – Kontrast komponentów UI (3:1) Obramowania kart: #3a3a3a na #141414 → 3.2:1. Ikony 3:1+.
✓
2.1.1 – Klawiatura Wszystkie karty mają tabindex="0". Skróty klawiszowe ↑↓/Enter/C widoczne w statusbarze.
✓
2.4.1 – Pomiń bloki Skip link "Przejdź do treści" widoczny po Tab na górze strony.
✓
2.4.3 – Kolejność focusu Logiczna kolejność: titlebar → toolbar → search → kafelki → statusbar.
✓
2.4.7 – Widoczny focus focus-visible: czerwony ring 3px rgba(255,0,0,.55) na wszystkich elementach interaktywnych.
✓
3.3.2 – Etykiety / instrukcje Wszystkie inputy mają etykiety (aria-label lub <label>). Radio w fieldset z legend.
✓
4.1.2 – Nazwa, rola, wartość Toggle: role="switch" aria-checked. Tabs: aria-selected. Karty: aria-label z tytułem.
✓
4.1.3 – Komunikaty statusu Liczba filmów i status synchronizacji mają aria-live="polite" aria-atomic="true".
